Complete Git Workflow for Team Development

1. Clone the Repository to Local

When a new developer joins the project, they need to clone the repository from the remote to their local machine: 

git clone <remote-repo-url>

This command will copy the entire project from the remote repository to your local machine.

2. Create a Feature Branch

Developers do not work directly on the main branch. Instead, they create separate branches for each feature or bug fix: 

git checkout main
git pull origin main
git checkout -b feature/awesome-feature

git checkout main switches to the main branch.
git pull origin main ensures the main branch is up to date.
git checkout -b feature/awesome-feature creates a new feature branch and switches to it.

3. Develop on the Feature Branch

Work on the feature branch and commit changes: 

git add <file-name>
git commit -m "Implemented awesome feature"

git add stages the modified files.
git commit commits the changes and describes what was changed.

4. Keep in Sync with Remote

Since others might be updating the codebase, you need to sync your branch periodically.

4.1 Fetch Updates from Remote

git fetch origin

This command fetches the latest updates from the remote repository without merging.

4.2 Merge the Latest Updates from Main

git merge origin/main

Ensure that your feature branch includes the latest changes from the main branch. If there are conflicts, resolve them and commit.

5. Push the Feature Branch to Remote

When the feature is ready for review, push the branch to the remote repository: 

git push origin feature/awesome-feature

This command creates a remote branch with the same name for team members to view, test, or review.

6. Create a Pull Request (PR)

After pushing, create a Pull Request (PR) on a platform like GitHub, GitLab, or Bitbucket to merge the feature branch into the main branch.

Team members can review the code and provide feedback.

7. Merge into Main Branch

Once the PR is approved, merge the feature branch into the main branch:

git checkout main
git pull origin main
git merge feature/awesome-feature
git push origin main

Switch to the main branch, ensure it is up to date, merge the feature branch, and push the updates to the remote repository.

8. Create a Version Tag

When the main branch reaches a stable state and is ready for release, create a tag:

8.1 Create an Annotated Tag

git tag -a v1.0.0 -m "Release version 1.0.0"

Create a tag named v1.0.0 with a description.

8.2 Push the Tag to Remote

git push origin v1.0.0

Push the tag to the remote repository so others can see it.

9. Delete the Feature Branch

Once the feature branch is merged and no longer needed, delete it:

9.1 Delete the Local Branch

git branch -d feature/awesome-feature

9.2 Delete the Remote Branch

git push origin --delete feature/awesome-feature

10. Return to Main Branch and Continue Working

After completing a feature, switch back to the main branch to get the latest updates and start new tasks:

git checkout main
git pull origin main

11. Manage Versions Using Tags

Tags help mark important milestones and releases. To switch to a particular release version:

git checkout v1.0.0

To delete a tag:

11.1 Delete a Local Tag

git tag -d v1.0.0

11.2 Delete a Remote Tag

git push origin --delete v1.0.0

Summary of Complete Workflow

  1. Clone the repository to local.
  2. Create a feature branch and develop on it.
  3. Push to remote and create a PR for review.
  4. Merge into the main branch after approval.
  5. Create version tags and push them to remote.
  6. Delete feature branches after merging.
  7. Use tags for version management and easy reference to releases.

This workflow helps ensure quality and traceability in team-based software development, with tags playing a crucial role in version management.
